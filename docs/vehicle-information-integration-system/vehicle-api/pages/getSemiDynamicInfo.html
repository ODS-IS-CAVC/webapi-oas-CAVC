準動的情報取得APIリクエストパラメータ詳細

準動的情報取得で利用するリクエストパラメータのJSONの内容を以下に記載する。

パラメータ

  • targetTime (string, date-time, 必須)
    • 説明: 要求対象のデータの日時
    • 例: 2024/01/01 00:00:00
  • requestInfo (array, 必須)
    • 説明: 検索するデータのリスト
    • 項目:
      • targetData (integer, int32)
        • 説明: 要求するデータを指定する。1:渋滞情報、2:事象規制情報、3:気象情報、4:待避エリア満空情報
        • 例: 1
      • requestFormat (integer, int32)
        • 説明: 要求対象範囲の形式を指定する。1:空間ID、2:緯度経度(bbox)
        • 例: 1
      • requestArea (object)
        • 説明: データの要求範囲を指定する。requestFormatで指定した形式の要求範囲を入力すること。
        • プロパティ:
          • spatialID (string)
            • 説明: 空間ID(z/f/x/y)
            • 例: 22/1/931369/413142
          • bbox (array)
            • 説明: バウンディングボックス（矩形領域座標）データモデル。「SW(経度), SW(緯度), NE(経度), NE(緯度)」
            • 例: [139.681994465, 35.50676814, 139.681985082, 35.506773265]

備考

  • requestInfoを同じtargetDataで複数要求し、そのrequestAreaが重複していた場合、応答情報に同一の情報が複数含まれる可能性があります
